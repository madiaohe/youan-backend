# 工作台页面设计文档

## 概述

在滤盒全生命周期管理系统中添加一个工作台（Dashboard）页面，作为用户进入系统后首先看到的首页。该页面汇总展示关键运营数据和待处理事项，帮助用户快速了解系统状态。

## 需求

### 用户需求
- 用户进入系统后，能快速了解今日运营状况
- 能看到近期检测、回收、发放趋势
- 能一目了然看到需要处理的事项

### 功能需求
1. 顶部 KPI 卡片：展示 4 个关键指标
2. 中间图表：检测/回收/发放趋势图，支持日/月/年切换
3. 底部表格：待处理事项列表

### 3. KPI 卡片详细设计

| 卡片 | 数据来源 | 说明 |
|------|----------|------|
| 今日检测数 | `mockDetectionStats` 今日数据 | 显示今日检测总数，对比昨日变化百分比 |
| 合格率 | 今日合格数/今日总数 | 显示百分比，对比昨日变化 |
| 待回收数 | `mockPendingRecycles` | 待回收滤盒总数 |
| 设备在线率 | `mockDetectorDevices` + `mockDispenserDevices` | 在线设备数/总设备数 |

### 4. 图表详细设计

**组件**: 使用 recharts + shadcn chart 组件

**数据结构**:
```typescript
interface ChartData {
  date: string;        // 日期标签
  detection: number;   // 检测数量
  recycle: number;     // 回收数量
  dispense: number;    // 发放数量
}
```

**交互**:
- 三个切换按钮：日 / 月 / 年
- 日视图：显示最近 7 天数据
- 月视图：显示最近 6 个月数据
- 年视图：显示最近 3 年数据

**图表类型**: 面积图（Area Chart）或折线图（Line Chart）

**颜色方案**:
- 检测数量：蓝色 `var(--chart-1)`
- 回收数量：橙色 `var(--chart-2)`
- 发放数量：绿色 `var(--chart-3)`

### 5. 待处理事项表格详细设计

**数据来源**:
1. 待回收滤盒：`mockPendingRecycles`
2. 设备告警：`mockSensorStatuses` 中状态为"异常"的
3. 库存预警：`mockDispenserDevices` 中库存低于阈值的

**表格字段**:
| 字段 | 说明 |
|------|------|
| 类型 | 待回收 / 设备告警 / 库存预警 |
| 内容 | 具体描述 |
| 时间 | 发生时间 |
| 操作 | 查看/处理/补货 按钮 |

**限制**: 显示最近 10 条记录

## 实现步骤

### 步骤 1: 安装依赖
```bash
npm install recharts
npx shadcn@latest add chart
```

### 步骤 2: 修改侧边导航
- 文件: `components/dashboard/app-sidebar.tsx`
- 添加"工作台"菜单项到顶部

### 步骤 3: 创建图表组件
- 文件: `components/dashboard/chart-trend.tsx`
- 实现趋势图，支持日/月/年切换

### 步骤 4: 修改首页
- 文件: `app/(dashboard)/page.tsx`
- 重写为工作台页面，包含 KPI + 图表 + 表格

### 步骤 5: 更新布局面包屑
- 文件: `app/(dashboard)/layout.tsx`
- 添加工作台的路由映射

## 技术约束

1. 必须使用 shadcn/ui 组件，不允许自定义组件（除了组合 shadcn 组件）
2. 图表使用 recharts + shadcn chart 封装
3. 遵循项目现有的代码风格和目录结构
4. 使用已有的 mock 数据，不新增 mock 数据结构

## 验收标准

1. 侧边栏顶部显示"工作台"菜单项
2. 点击"工作台"进入首页，显示 KPI 卡片、图表、表格
3. 图表支持日/月/年切换，数据正确显示
4. 待处理事项表格显示待回收、设备告警、库存预警数据
5. 页面布局与设计稿一致
